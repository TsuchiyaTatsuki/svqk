= アーキテクチャ仕様書

本書の対象読者は、本システムの開発・保守における実装、自動テスト、CI・CD構築を担当する開発者です。また本書は、本システム、およびその開発・保守における、アプリケーション、開発・CI・CD環境、自動テストの側面に対する記載です。

本書の目的は、開発者が各側面において作成する成果物、主にアプリケーションのソースコード、テストスクリプト、ビルドスクリプト、設定ファイルを作成する際の規則を定義することです。規則化の対象となるのは、ソースコードをファイルに記載する際の、記載内容の複数ファイルへの分割、ファイル内でのプログラミング言語の構文に則った構成要素への分割、ファイルの配置ディレクトリ、それらの命名等です。このソースコードの記載内容の分割において、同種の記載内容の集合を責務と呼び、同種の責務を割り当てられた集合をコンポーネントと呼びます。コンポーネントの構成、及びそれらの相互作用を可視化することで、開発者による規則の理解を助けることを目指します。
